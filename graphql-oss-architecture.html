Tool Deep Dives
The OSS Tools — What They Are & Why They Win
Each tool was chosen because it directly replaces a proprietary Apollo product, is actively maintained, has a strong community, and can be fully self-hosted.
🐝

GraphQL Hive

MIT · github.com/kamilkisiela/graphql-hive

Replaces: Apollo GraphOS + Apollo Studio

Full schema registry with SDL versioning, breaking change history, field usage tracking (know which fields to deprecate safely), and a schema explorer UI — all self-hosted via Docker Compose.

The Hive CLI replaces rover for publishing and checking subgraph schemas in CI. No Apollo account, no API keys to Apollo's cloud. Works completely offline.

schema registryfield analyticsCI checkself-hostedDocker Compose
🔍

graphql-inspector

MIT · github.com/kamilkisiela/graphql-inspector

Replaces: rover subgraph check

CLI that diffs two SDL files and classifies every change as SAFE, DANGEROUS, or BREAKING. Fails the CI pipeline on breaking changes. No account required — runs entirely from the SDL file on disk.

GitHub Action available. Supports custom rules. Can validate operations against a schema (catches client-side breaks too).

breaking change CIno accountofflineGitHub ActionSDL diff

The Guild Conductor

Apache 2.0 · the-guild.dev/conductor

Replaces: Apollo Router (paid features)

A fully open-source GraphQL gateway with federation support, plugin system, rate limiting, APQ (persisted queries), JWT auth, and request deduplication — ALL features included, no SaaS plan required.

Single binary or Docker container. Config via YAML. Actively maintained by The Guild (same team that maintains graphql-yoga, graphql-tools, graphql-inspector).

federation gatewayAPQ built-inrate limitingJWT authYAML config
🔭

OpenTelemetry + Jaeger

Apache 2.0 · CNCF graduated

Replaces: Apollo Studio Tracing

OTEL instruments every resolver via @envelop/opentelemetry plugin. Each resolver call emits a span with timing, errors, and field name. Jaeger collects these spans and renders resolver flame graphs — identical UX to Apollo Studio tracing, fully self-hosted.

resolver flame graphself-hostedCNCFk8s nativeGDPR safe
🧘

graphql-yoga + envelop

MIT · the-guild.dev/graphql-yoga

Replaces: @apollo/server + @apollo/subgraph

graphql-yoga is a fully featured GraphQL server (Node.js, Deno, Bun, Edge). envelop is a plugin system that adds auth, tracing, caching, rate limiting, and depth limiting without touching resolver code.

Federation spec support via @envelop/federation. Zero Apollo dependency.

server runtimeplugin systemfederationedge runtimezero Apollo
📊

Prometheus + Grafana Stack

Apache 2.0 + AGPL

Replaces: Apollo GraphOS dashboards + PagerDuty

Prometheus scrapes custom metrics from Conductor (GQL ops/sec, cache hit rate, error rate per subgraph). Grafana renders SLA dashboards with P50/P95/P99 latency panels. Alertmanager handles on-call routing — same as PagerDuty but self-hosted.

# .github/workflows/schema-check.yml
# Uses graphql-inspector (MIT) — NO Apollo account needed

name: GraphQL Schema Check
on: [pull_request]

jobs:
  schema-diff:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }

      # Install graphql-inspector CLI (MIT license)
      - run: npm install -g @graphql-inspector/cli

      # Diff absence subgraph against main branch SDL
      - name: Check absence subgraph for breaking changes
        run: |
          git show origin/main:absence/schema.graphql > /tmp/old-absence.graphql
          graphql-inspector diff \
            /tmp/old-absence.graphql \
            absence/schema.graphql \
            --fail-on-breaking-change
          # EXIT 1 on BREAKING changes · passes on SAFE additions

      - name: Check balances subgraph
        run: |
          git show origin/main:balances/schema.graphql > /tmp/old-bal.graphql
          graphql-inspector diff /tmp/old-bal.graphql balances/schema.graphql \
            --fail-on-breaking-change

      - name: Check history subgraph
        run: |
          git show origin/main:history/schema.graphql > /tmp/old-hist.graphql
          graphql-inspector diff /tmp/old-hist.graphql history/schema.graphql \
            --fail-on-breaking-change

      # Also validate all client operations against new schema
      - name: Validate client operations
        run: |
          graphql-inspector validate \
            'src/**/*.graphql' \
            absence/schema.graphql

      # Publish SDL to self-hosted Hive (replaces rover push)
      - name: Publish schema to Hive registry
        if: github.ref == 'refs/heads/main'
        run: |
          hive schema:publish absence/schema.graphql \
            --service absence \
            --url http://absence-svc:4001/graphql
        env:
          HIVE_TOKEN: ${{ secrets.HIVE_TOKEN }}  # your own Hive instance
          HIVE_REGISTRY: http://hive.internal:3000  # self-hosted
// absence-server.ts — graphql-yoga (MIT) + envelop plugins (MIT)
// Replaces @apollo/server entirely
import { createYoga } from 'graphql-yoga'
import { useOpenTelemetry } from '@envelop/opentelemetry'
import { useDepthLimit } from '@envelop/depth-limit'
import { useRateLimiter } from '@envelop/rate-limiter'
import { useHive } from '@graphql-hive/client'
import { schema } from './schema'

const yoga = createYoga({
  schema,
  plugins: [
    // OTEL tracing → Jaeger (replaces Apollo Studio tracing)
    useOpenTelemetry({
      resolvers: true,      // per-resolver spans
      variables: true,      // log query variables
      result: false,        // don't log full result (PII)
    }),

    // Depth limit (replaces Apollo Router paid feature)
    useDepthLimit({ maxDepth: 8 }),

    // Field usage reporting → self-hosted Hive
    // (replaces Apollo GraphOS usage analytics)
    useHive({
      enabled: true,
      token: process.env.HIVE_TOKEN,   // your own Hive instance
      reporting: {
        endpoint: 'http://hive.internal:3000/registry/internal',
        author: 'absence-service',
        commit: process.env.GIT_SHA,
      },
      usage: {
        endpoint: 'http://hive.internal:3000/usage',
        sampleRate: 1.0,     // 100% of operations tracked
      },
    }),

    // Rate limiting per authenticated user
    useRateLimiter({
      identifyFn: (ctx) => ctx.user?.id ?? ctx.req.ip,
      config: { window: '1s', max: 100 },
    }),
  ],
})

export { yoga }

  resource: new Resource({
    [SEMRESATTRS_SERVICE_NAME]: 'absence-subgraph',

  traceExporter: new OTLPTraceExporter({
    url: 'http://jaeger:4317',  // self-hosted Jaeger

})
sdk.start()
