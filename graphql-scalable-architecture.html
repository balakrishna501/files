Production Pillars
6 Pillars of Robust GraphQL
Each pillar addresses a specific failure mode in the original architecture. Together they form a system that is operationally mature — not just technically correct.
⟳

Horizontal Scaling

Every component scales independently. Router pods autoscale on CPU. Subgraph pods autoscale on request queue depth. DBs scale read traffic via replicas.

  • Apollo Router: ×3 pods minimum, HPA to ×20
  • Each subgraph: ×2 pods minimum, independent HPA
  • DB read replicas: ×2 per domain, all reads go to replicas
  • Redis cluster: ×3 nodes with sharding
  • Kubernetes HPA on custom metrics (GQL ops/sec)
⚡

Fault Isolation & Circuit Breaking

One failing subgraph must never cascade to take down the entire gateway. Circuit breakers at the gateway open on error rate threshold and return partial data with graceful degradation.

  • Per-subgraph circuit breaker in Apollo Router
  • Partial response: Absence fails → balance + history still return
  • Timeout per subgraph: 2s max, not the global request timeout
  • Bulkhead pattern: separate thread pools per subgraph
  • Retry with exponential backoff (mutations excluded)
◉

Deep Observability

You cannot improve what you cannot measure. Field-level tracing from Apollo Studio gives resolver flame graphs. OTEL spans flow from edge CDN all the way to DB query, connected by trace ID.

  • OpenTelemetry spans per resolver — Jaeger / Tempo
  • Apollo Studio field usage: know which fields to deprecate safely
  • Prometheus: custom metrics — GQL ops/sec, cache hit rate, N+1 detections
  • PagerDuty: alert on P95 > 200ms or error rate > 1%
  • Grafana SLA dashboard: real-time 99.99% uptime tracking
🔒

Security Hardening

GraphQL's expressive query language is a larger attack surface than REST. Multiple layers of defence — from the WAF at the edge down to field-level authorization inside resolvers.

  • WAF + DDoS protection at CDN (Cloudflare)
  • JWT validation in Router before any resolver fires
  • Query depth limit (8) + complexity budget (500)
  • Field-level @auth directives — balance only visible to owner
  • Introspection disabled in production
  • mTLS between Router and subgraphs
⚡

Multi-Layer Caching

Caching at every layer eliminates redundant work. A balance query answered from CDN costs zero server compute. Even when it misses CDN, Redis answers before any DB call fires.

  • CDN edge: GET queries cacheable via APQ hash URL
  • Redis: @cacheControl TTL per field type (balance: 30s)
  • DataLoader: per-request batching eliminates N+1
  • DB read replicas: read-heavy queries never touch primary
  • Apollo persisted queries: 90% bandwidth reduction on mobile
◈

Developer Experience & CI/CD

The schema is the API contract. Every breaking change must be caught before production — not discovered by a mobile app at runtime. The CI pipeline is the enforcer.

  • graphql-codegen: TypeScript types from SDL — compile-time safety
  • rover subgraph check: CI pipeline gate for breaking changes
  • Blue/green deploys: zero-downtime subgraph updates
  • Canary rollout: 5% → 25% → 100% with auto-rollback
  • Schema changelog in Apollo Studio: full change history
Resilience Engineering
Critical Patterns Not in the Original Design
These patterns are the difference between a system that works in demos and one that works at 3am during a production incident.
⚡

Circuit Breaker — Per Subgraph

Prevents cascade failure

If Balances subgraph error rate exceeds 50% in a 10s window, the circuit opens. Subsequent requests get a partial response (absence + history only) in under 10ms rather than waiting 2s for a timeout. After 30s, the circuit probes with 1 request — if it succeeds, it closes.

Without this, a single slow subgraph holds all gateway threads until the whole router dies.

half-open probe@fallback directivepartial responseerror threshold 50%
↺

Async Mutations via Event Bus

Kafka / SQS with DLQ

Time-off submissions and approvals are mutations that trigger downstream effects (email, payroll, calendar sync). Instead of calling each synchronously (fragile), the mutation publishes to Kafka. Each consumer processes independently with exponential-backoff retry.

If a consumer fails 3 times, the message goes to a Dead Letter Queue for manual inspection — not silently lost.

at-least-once deliveryexponential backoffDLQidempotency key
◐

Blue/Green + Canary Subgraph Deploys

Zero-downtime releases

New subgraph versions deploy alongside the old. The Apollo Router progressively shifts traffic: 5% → 25% → 100% over 30 minutes. If the new version's error rate exceeds the baseline by 2×, traffic shifts back automatically — the developer gets a Slack alert, not a 3am call.

traffic splittingauto-rollbackerror rate baselinezero downtime
⊕

Multi-AZ Deployment

3 availability zones

Router pods, subgraph pods, Redis nodes, and DB read replicas are spread across 3 availability zones using Kubernetes pod anti-affinity rules. An entire AZ going dark — which happens during AWS/GCP incidents — drops only 1/3 of capacity, not the whole service.

// subgraph_request.rhai
fn subgraph_request(request) {
  let subgraph = request.subgraph.name;
  let error_rate = get_metric(`subgraph_error_rate{name=${subgraph}}`);

  if error_rate > 0.5 {
    // Circuit is OPEN — return partial response immediately
    return ControlFlow::Fail(
      Response::error_builder()
        .errors(vec![GraphQLError {
          message: `${subgraph} temporarily unavailable`,
          extensions: map! { "code": "CIRCUIT_OPEN" }
        }])
        .build()
    );
  }
  ControlFlow::Continue(request)
}

// In resolvers — handle partial failures gracefully
const resolvers = {
  Query: {
    employee: async (_, { id }, ctx) => {
      const [absence, balance] = await Promise.allSettled([
        ctx.absence.load(id),
        ctx.balance.load(id),
      ])
      // Return what we have — not all or nothing
      return {
        id,
        absences: absence.status === 'fulfilled' ? absence.value : null,
        balance: balance.status === 'fulfilled' ? balance.value : null,
      }
    }
  }
}
// context.ts — create DataLoaders per request (not global!)
export function createContext({ req }: { req: Request }) {
  const employeeId = verifyJWT(req.headers.authorization)?.sub

  return {
    employeeId,
    loaders: {
      // One DataLoader per data type — never share across requests
      balance: new DataLoader(async (ids: readonly string[]) => {
        const rows = await db.query(
          'SELECT * FROM balances WHERE employee_id = ANY($1) AND tenant_id = $2',
          [ids, getTenantId(req)]   // tenant isolation!
        )
        return ids.map(id => rows.find(r => r.employee_id === id)
          ?? new Error(`Balance not found: ${id}`))
      }, {
        cache: true,         // dedupe within request
        maxBatchSize: 100,   // split if > 100 IDs
        batchScheduleFn: callback => setTimeout(callback, 0)
      }),

      absence: new DataLoader(async (ids: readonly string[]) => {
        const rows = await absenceDb.query(
          'SELECT * FROM absences WHERE employee_id = ANY($1)', [ids]
        )
        // Group by employee_id — one employee has many absences
        return ids.map(id => rows.filter(r => r.employee_id === id))
      })
    }
  }
}
// Subscription — real-time balance push via Redis PubSub
import { RedisPubSub } from 'graphql-redis-subscriptions'

const pubsub = new RedisPubSub({
  publisher: new Redis({ host: 'redis-0', port: 6379 }),
  subscriber: new Redis({ host: 'redis-0', port: 6379 }),
})

const resolvers = {
  Mutation: {
    approveAbsence: async (_, { id }, ctx) => {
      // 1. Update DB
      const updated = await db.approveAbsence(id)
      // 2. Recalculate balance
      const newBalance = await db.recalcBalance(updated.employee_id)
      // 3. Publish to Redis — subscribers get push in <100ms
      await pubsub.publish(
        `BALANCE_UPDATED:${updated.employee_id}`,
        { balanceUpdated: newBalance }
      )
      // 4. Also emit to event bus for async downstream effects
      await kafka.produce('absence-approved', updated)
      return updated
    }
  },
  Subscription: {
    balanceUpdated: {
      subscribe: (_, { employeeId }, ctx) => {
        // Security: can only subscribe to OWN balance
        if (ctx.employeeId !== employeeId) throw new Error('Forbidden')
        return pubsub.asyncIterator(`BALANCE_UPDATED:${employeeId}`)
      }
    }
  }
}
